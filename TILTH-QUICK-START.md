## ğŸ­ Understanding the Two Modes

### Product Problems Mode
**What you design:** The product itself  
**Your role:** External designer with authority  
**Focus:** Ethical alternatives to harmful artifacts  
**Example output:** Redesigned learning platform without addictive mechanics  
**Tetrad applied to:** Product features and affordances  

### Designer Dilemmas Mode  
**What you design:** Organizational infrastructure  
**Your role:** The actual designer from the scenario (limited power)  
**Focus:** Systemic solutions to second-order problems  
**Example output:** Metrics system that makes learning valuable vs. engagement  
**Tetrad applied to:** Governance, processes, and organizational systems  

### When to Use Each Mode

| Use Product Problems when... | Use Designer Dilemmas when... |
|------------------------------|-------------------------------|
| You want to explore ethical product design | You want to explore organizational change |
| Focus is on user experience | Focus is on institutional dynamics |
| Question: "What should this be?" | Question: "How do we change the system?" |
| Output: Working prototype | Output: Governance/metrics/process system |
| Stakeholders: End users | Stakeholders: Internal + power structures |
| Design challenge: Technical/UX | Design challenge: Political/systemic |

### The Same Scenario, Different Lenses

**GrowthPath Example:**

**Product Problems:**
- Design a non-addictive learning platform

- Output: Educational app prototype
- Success: Users learn without addiction

**Designer Dilemmas:**
- YOU are Maya Chen, L&D Lead

- Design measurement system valuing learning outcomes
- Output: Dashboard + methodology for leadership
- Success: Organization shifts from engagement to learning value

**Both are valid design challenges!** One focuses on the artifact, the other on the conditions that created it.

## ğŸ¨ Design Philosophy

Each TILTH scenario is structured to:

1. **Describe the current system** â€” What exists now
2. **Surface the ethical tension** â€” The core conflict
3. **Name the power dynamic** â€” Who's pressuring whom
4. **Set the design challenge** â€” What needs redesigning

The Tetrad framework helps you explore **four facets** of each problem:
- What to **amplify** (Enhance)
- What to **remove** (Obsolesce)
- What to **restore** (Retrieve)
- What **reverses** at overextension (Reverse)
